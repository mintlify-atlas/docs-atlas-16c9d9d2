## BlockNote Editor

The editor is built on [BlockNote](https://www.blocknotejs.org/), a modern block-based editor:

### Block-Based Editing

Everything in the editor is a block:
- Paragraphs
- Headings (H1-H6)
- Lists (ordered and unordered)
- Code blocks
- Blockquotes

```typescript
// Blocks are parsed from markdown on load
// From: components/blocknote-editor.tsx:25-31
if (editor && value) {
  const loadContent = async () => {
    const blocks = await editor.tryParseMarkdownToBlocks(value);
    editor.replaceBlocks(editor.document, blocks);
  };
  loadContent();
}
```

## Autosave

Changes are automatically saved as you type:

### Save Triggers

1. **Automatic**: Changes trigger autosave after a short delay
2. **Manual**: Press `⌘S` to save immediately
3. **Before navigation**: Saves before switching files

```typescript
// Manual save with keyboard shortcut
// From: components/app-shell.tsx:136-143
if (e.key === "s" && (e.metaKey || e.ctrlKey)) {
  e.preventDefault();
  const editorState = useEditorStore.getState();
  if (editorState.fileKey && editorState.dirty && 
      editorState.status !== "saving") {
    void editorState.save("manual");
  }
}
```

### Save States

The editor shows different states in the footer:

| State | Description | Action |
|-------|-------------|--------|
| `Saved • [timestamp]` | All changes saved | None needed |
| `Unsaved changes` | Changes pending save | Wait for autosave or press ⌘S |
| `Saving...` | Save in progress | Wait for completion |
| `Failed to save` | Save error occurred | Click "Retry" to try again |
| `Save blocked by remote changes` | Conflict detected | Click "Reload" to fetch latest |

<Warning>
  If you navigate away with unsaved changes, the browser will prompt you to confirm. This prevents accidental data loss.
</Warning>

### Conflict Resolution

If the file was modified externally while you were editing:

1. The editor detects the conflict using ETag validation
2. Save is blocked to prevent overwriting remote changes
3. You see "Save blocked by remote changes" in the footer
4. Click **Reload** to fetch the latest version (discards your changes)
5. Or manually merge changes by copying your edits first

## Editor Integration

The editor integrates with the editor store for state management:

```typescript
// Editor store integration
// From: components/blocknote-editor.tsx:17-18
const registerEditorView = useEditorStore((state) => state.registerEditorView);
const setSelection = useEditorStore((state) => state.setSelection);
```

### Content Synchronization

Changes flow through the store:

1. User types in the editor
2. `onChange` handler converts blocks to markdown
3. Markdown is stored in the editor state
4. Autosave triggers after debounce
5. Changes are persisted to S3

```typescript
// Change handler converts blocks to markdown
// From: components/blocknote-editor.tsx:104-108
const handleChange = async () => {
  if (!editor) return;
  const markdown = await editor.blocksToMarkdownLossy(editor.document);
  onChange(markdown);
};
```

## Reading Time

The editor calculates estimated reading time shown in the footer:

```typescript
// Reading time calculation
// From: components/app-shell.tsx:813-830
function computeReadingTimeLabel(content: string, hasFile: boolean): string {
  if (!hasFile) return "—";
  
  const words = content.trim().split(/\s+/).filter(Boolean).length;
  if (!words) return "< 1 min";
  
  const minutes = words / 200; // 200 words per minute
  if (minutes < 1) return "< 1 min";
  if (minutes < 60) return `${Math.round(minutes)} min`;
  
  const hours = minutes / 60;
  return `${hours.toFixed(1)} hr`;
}
```

Reading time assumes 200 words per minute and appears in the bottom-right footer.

The editor is optimized for large documents:

- Block-based rendering only shows visible content
- Debounced save prevents excessive API calls
- Efficient markdown parsing with lossy conversion
- Lazy loading for preview mode
