### Search Behavior

The search filter uses smart matching:

```typescript
// Match logic includes parent folders
// From: components/file-tree/index.tsx:176-197
const matchMap = useMemo(() => {
  if (!filterActive) return null;
  
  const map = new Map<NodeId, MatchMeta>();
  const visit = (id: NodeId): boolean => {
    const node = nodes[id];
    if (!node) return false;
    
    const matchesSelf = node.name.toLowerCase().includes(normalizedQuery);
    let hasMatchingChild = false;
    
    if (node.type === "folder") {
      hasMatchingChild = node.children.some((childId) => visit(childId));
    }
    
    const include = matchesSelf || hasMatchingChild;
    map.set(id, { include, matchesSelf, hasMatchingChild });
    return include;
  };
  
  rootIds.forEach((id) => visit(id));
  return map;
}, [filterActive, nodes, normalizedQuery, rootIds]);
```

<Accordion title="Search Features">
  **Matches**
  - File names (e.g., "meeting" finds "Meeting Notes.md")
  - Folder names (e.g., "projects" finds "Projects/")
  - Partial matches (e.g., "proj" finds "Projects")
  
  **Parent Folders**
  - Folders containing matching files are shown
  - Automatically expanded to reveal matches
  - Full path context preserved
  
  **Performance**
  - 200ms debounce prevents excessive filtering
  - Results cached until query changes
  - Efficient tree traversal algorithm
</Accordion>

```typescript
      return;

      const editorState = useEditorStore.getState();
      if (editorState.fileKey && editorState.dirty) {
        void editorState.save("manual");
      }
      return;
    }
  };
  
  window.addEventListener("keydown", onKey);
  return () => window.removeEventListener("keydown", onKey);
}, []);
```

### Breadcrumb Building

Breadcrumbs are constructed from the file path:

```typescript
// Convert path to breadcrumb segments
// From: components/vault-workspace/index.tsx:545-557
function buildSegments(path: string): BreadcrumbSegment[] {
  const cleaned = path.endsWith("/") ? path.slice(0, -1) : path;
  if (!cleaned) return [];
  
  const parts = cleaned.split("/");
  return parts.map((label, index) => {
    const isLast = index === parts.length - 1;
    // Build cumulative path for parent folders
    const cumulativePath = isLast 
      ? null 
      : parts.slice(0, index + 1).join("/") + "/";
    return { label, path: cumulativePath };
  });
}
```

Example:
```
Projects/2024/Q1 Goals.md
↓
[
  { label: "Projects", path: "Projects/" },
  { label: "2024", path: "Projects/2024/" },
  { label: "Q1 Goals", path: null }
]
```

## Sibling Navigation

Navigate between files in the same folder:

### Previous/Next Buttons

Arrow buttons in the header (desktop only) move between siblings:

```typescript
// Sibling navigation logic
// From: components/vault-workspace/index.tsx:154-188
const siblingNavigation = useMemo(() => {
  if (!selectedId) return { prevId: null, nextId: null };
  
  const selectedNode = nodes[selectedId];
  if (!selectedNode || selectedNode.type !== "file") {
    return { prevId: null, nextId: null };
  }
  
  const parentId = selectedNode.parentId;
  if (!parentId) return { prevId: null, nextId: null };
  
  const parentNode = nodes[parentId];
  if (!parentNode || parentNode.type !== "folder") {
    return { prevId: null, nextId: null };
  }
  
  // Get only file siblings (exclude folders)
  const fileSiblingIds = parentNode.children
    .filter((childId) => nodes[childId]?.type === "file");
  
  if (fileSiblingIds.length <= 1) {
    return { prevId: null, nextId: null };
  }
  
  const currentIndex = fileSiblingIds.indexOf(selectedNode.id);
  return {
    prevId: currentIndex > 0 ? fileSiblingIds[currentIndex - 1] : null,
    nextId: currentIndex < fileSiblingIds.length - 1 
      ? fileSiblingIds[currentIndex + 1] 
      : null,
  };
}, [nodes, selectedId]);
```

<Tip>
  Sibling navigation only considers files in the same parent folder, making it easy to review related notes sequentially.
</Tip>

## Performance Optimizations

### Debounced Search

Search input is debounced to prevent excessive re-renders:

```typescript
// 200ms debounce for search
const debouncedQuery = useDebouncedValueHook(query, 200);
```

### Memoized Filtering

Search results are memoized to avoid recalculation:

```typescript
// Memoized match map only recalculates when dependencies change
const matchMap = useMemo(() => {
  // ... filtering logic
}, [filterActive, nodes, normalizedQuery, rootIds]);
```

### Efficient Tree Traversal

The search algorithm visits each node only once:

```typescript
// Single-pass tree traversal
const visit = (id: NodeId): boolean => {
  const node = nodes[id];
  const matchesSelf = node.name.toLowerCase().includes(normalizedQuery);
  const hasMatchingChild = node.type === "folder" 
    ? node.children.some((childId) => visit(childId))
    : false;
  return matchesSelf || hasMatchingChild;
};
```
