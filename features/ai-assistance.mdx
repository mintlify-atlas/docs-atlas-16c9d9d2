## AI Result Panel

When you trigger an AI action, results appear in a panel above your note:

### Panel Features

<AccordionGroup>
  <Accordion title="Real-time Streaming" icon="stream">
    AI responses stream in real-time as they're generated:
    
    - See partial results immediately
    - Loading indicator shows streaming progress
    - Cancel button available during generation
    
    ```typescript
    // Streaming state management
    const isStreaming = status === "submitted" || status === "streaming";
    ```
  </Accordion>
  
  <Accordion title="Result Actions" icon="bolt">
    Once generation completes, you can:
    
    **Insert** - Add AI result at cursor position
    ```typescript
    // Insert at cursor
    applyAiResult(text, { strategy: "insert" });
    ```
    
    **Replace** - Replace selected text or entire content
    ```typescript
    // Replace selection or document
    applyAiResult(text, { strategy: "replace" });
    ```
    
    **Copy** - Copy to clipboard for use elsewhere
    
    **Retry** - Generate a new response with the same prompt
  </Accordion>
  
  <Accordion title="Error Handling" icon="triangle-exclamation">
    If generation fails:
    
    - Clear error message displayed
    - Retry button available
    - Can close panel to dismiss
    
    Common errors include rate limits, network issues, or content policy violations.
  </Accordion>
</AccordionGroup>

### Insert into Editor

When you click Insert:

1. The chat sidebar remains open
2. Content is added to your active note
3. Editor switches to edit mode if in preview
4. Toast notification confirms insertion

```typescript
// Insert handler
// From: components/ai-chat/sidebar-chat.tsx:684-701
const handleInsert = useCallback(
  (message: UIMessage) => {
    const text = messageToPlainText(message);
    if (!text.trim()) {
      toast({
        title: "No content",
        description: "Wait for the assistant response before inserting.",
      });
      return;
    }
    applyAiResult(text, { strategy: "insert" });
    toast({
      title: "Inserted",
      description: "AI response added to the editor.",
    });
  },
  [applyAiResult, toast],
);
```

## AI State Management

The AI system uses a state machine for reliable operation:

```typescript
// AI session states: idle → loading → streaming → success/error
// Managed by useAiSession hook
const {
  state: aiState,
  panelOpen,
  isStreaming,
  start,      // Trigger action
  cancel,     // Stop generation
  retry,      // Retry failed request
  closePanel, // Dismiss panel
  applyReplace,
  applyInsert,
} = useAiSession({ /* ... */ });
```

States include:
- `idle`: No active operation
- `loading`: Request sent, waiting for first token
- `streaming`: Receiving response
- `success`: Generation complete
- `error`: Generation failed

## Limitations

<Warning>
  AI features require:
  - Active internet connection
  - Valid AI Gateway configuration
  - At least one word of content in the editor
  - Available API quota/credits
</Warning>

### When AI is Disabled

AI actions are disabled when:
- No file is open
- File is still loading
- Document is empty
- Another AI operation is in progress
- Save conflict exists

```typescript
// AI disabled conditions
// From: components/vault-workspace/index.tsx:388
aiDisabled={!hasFile || status === "loading" || 
           !hasDocumentContent || aiStreaming}
```

## Privacy & Data

- Content is sent to AI providers for processing
- File content includes up to 6,000 characters as context
- Chat messages are not persisted server-side
- Model selection is cached locally for 12 hours
