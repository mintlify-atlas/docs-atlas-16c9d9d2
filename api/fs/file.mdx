## Get File

```
GET /api/fs/file
```

Retrieves the content and metadata of a markdown file. Supports conditional requests via `If-None-Match` and `If-Modified-Since` headers.

### Query Parameters

<ParamField query="key" type="string" required>
  The file path relative to the vault root. Must end with `.md` extension.
  
  **Validation:**
  - Must be relative (no leading `/`)
  - Must use `/` as separator (not `\`)
  - Cannot contain `..` segments
  - Must end with `.md`
</ParamField>

### Request Headers

<ParamField header="If-None-Match" type="string" optional>
  Comma-separated list of ETags. If the file's current ETag matches any provided value, returns `304 Not Modified`.
</ParamField>

<ParamField header="If-Modified-Since" type="string" optional>
  HTTP date string. If the file hasn't been modified since this date, returns `304 Not Modified`.
</ParamField>

### Response

<ResponseField name="key" type="string">
  The file path that was requested.
</ResponseField>

<ResponseField name="content" type="string">
  The markdown file content.
</ResponseField>

<ResponseField name="etag" type="string" optional>
  The current ETag identifier for the file version.
</ResponseField>

<ResponseField name="lastModified" type="string" optional>
  ISO 8601 timestamp of when the file was last modified.
</ResponseField>

### Response Headers

- `ETag`: The file's current ETag (quoted)
- `Last-Modified`: HTTP date when file was last modified
- `Cache-Control`: `private, no-cache, must-revalidate`
- `X-File-Cache`: Cache status (`HIT`, `MISS`, or `STALE`)

### Examples

```bash
curl -X GET "https://your-app.com/api/fs/file?key=notes/todo.md"
```

```javascript
const response = await fetch('/api/fs/file?key=notes/todo.md');
const data = await response.json();

console.log(data.content); // "# My Todo List\n- Task 1\n..."
console.log(data.etag);    // "abc123def456"
```

#### Conditional request

```javascript
const response = await fetch('/api/fs/file?key=notes/todo.md', {
  headers: {
    'If-None-Match': '"abc123def456"'
  }
});

if (response.status === 304) {
  console.log('File unchanged, use cached version');
}
```

## Update File

```
PUT /api/fs/file
```

Writes or updates a markdown file. Supports optimistic concurrency control via `ifMatchEtag`.

### Request Body

<ParamField body="key" type="string" required>
  The file path relative to the vault root. Must end with `.md` extension.
  
  **Validation:**
  - Must be relative (no leading `/`)
  - Must use `/` as separator (not `\`)
  - Cannot contain `..` segments
  - Must end with `.md`
</ParamField>

<ParamField body="content" type="string" required>
  The markdown content to write. Can be empty string for blank files.
</ParamField>

<ParamField body="ifMatchEtag" type="string" optional>
  Expected current ETag of the file. If provided and doesn't match the current version, returns `412 Precondition Failed`. Use for conflict-free updates.
</ParamField>

### Response

<ResponseField name="etag" type="string">
  The new ETag identifier for the updated file version.
</ResponseField>

### Examples

#### Create new file

```bash
curl -X PUT "https://your-app.com/api/fs/file" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "notes/new-note.md",
    "content": "# New Note\\n\\nContent here"
  }'
```

```javascript
const response = await fetch('/api/fs/file', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    key: 'notes/new-note.md',
    content: '# New Note\n\nContent here'
  })
});

const { etag } = await response.json();
console.log('File created with ETag:', etag);
```

#### Update with conflict prevention

```javascript
const response = await fetch('/api/fs/file', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    key: 'notes/todo.md',
    content: '# Updated Todo List\n- New task',
    ifMatchEtag: 'abc123def456' // Only update if this is current version
  })
});

if (response.status === 412) {
  console.error('File was modified by someone else');
}
```

## Delete File

```
DELETE /api/fs/file
```

Deletes a markdown file from the vault. Supports optimistic concurrency control via `ifMatchEtag`.

### Request Body

<ParamField body="key" type="string" required>
  The file path relative to the vault root. Must end with `.md` extension.
  
  **Validation:**
  - Must be relative (no leading `/`)
  - Must use `/` as separator (not `\`)
  - Cannot contain `..` segments
  - Must end with `.md`
</ParamField>

<ParamField body="ifMatchEtag" type="string" optional>
  Expected current ETag of the file. If provided and doesn't match, returns `409 Conflict`. Use to prevent accidental deletion of modified files.
</ParamField>

### Response

Returns `204 No Content` on success with an empty body.

### Examples

```bash
curl -X DELETE "https://your-app.com/api/fs/file" \
  -H "Content-Type: application/json" \
  -d '{"key": "notes/old-note.md"}'
```

```javascript
const response = await fetch('/api/fs/file', {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    key: 'notes/old-note.md'
  })
});

if (response.status === 204) {
  console.log('File deleted successfully');
}
```

#### Delete with conflict prevention

```javascript
const response = await fetch('/api/fs/file', {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    key: 'notes/old-note.md',
    ifMatchEtag: 'abc123def456'
  })
});

if (response.status === 409) {
  console.error('File was modified, cannot delete');
}
```

## Error Responses

### 400 Bad Request

Returned when request parameters are invalid:

```json
{
  "error": "File key is required"
}
```

Other validation errors:
- `"File key must be relative to the vault root"`
- `"File key must use '/' as a separator"`
- `"File key cannot contain '..'"` 
- `"Only markdown (.md) files are supported"`
- `"Bad Request"` (S3-level validation failure)

### 401 Unauthorized

```json
{
  "error": "Unauthorized"
}
```

### 404 Not Found

Returned when the requested file doesn't exist:

```json
{
  "error": "File not found"
}
```

### 409 Conflict

Returned when `ifMatchEtag` doesn't match current version (DELETE) or ETag mismatch (internal):

```json
{
  "error": "ETag mismatch"
}
```

### 412 Precondition Failed

Returned when `ifMatchEtag` doesn't match current version (PUT):

```json
{
  "error": "Precondition failed"
}
```

### 500 Internal Server Error

Returned when S3 operations fail:

```json
{
  "error": "Failed to process request"
}
```
